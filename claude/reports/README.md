# Reports Directory

This directory contains reports generated by various agents in the OpenSlots project. Reports are organized by agent area with consistent naming conventions.

---

## Directory Structure

```
claude/reports/
├── README.md           (this file)
├── testing/            (from test-runner agent)
│   └── test-runs.md    (summary of all test executions)
└── UAT/                (User Accessibility Tests from ua-tester agent)
    ├── UAT-2025-11-27-v001.md
    ├── UAT-2025-11-27-v002.md
    └── UAT-2025-11-28-v001.md
```

---

## Naming Conventions

All reports follow a consistent naming pattern:

```
<AGENT>-YYYY-MM-DD-vNN.md
```

Where:
- `<AGENT>` = Agent type (UAT, testing, etc.)
- `YYYY-MM-DD` = Date of report creation
- `vNN` = Version number (2 digits: v01, v02, etc.)
- Version resets daily (first report each day is v01)

**Examples:**
- `UAT-2025-11-28-v01.md` (first UAT report on Nov 28)
- `UAT-2025-11-28-v02.md` (second UAT report same day)
- `UAT-2025-11-29-v01.md` (new day, version resets)

---

## Report Metadata

Every report file must include metadata at the top:

```markdown
# Report Title

**Report ID:** <AGENT>-YYYY-MM-DD-vNN
**Date:** YYYY-MM-DD
**Time:** HH:MM:SS +TIMEZONE
**Agent:** <agent-name>
**Type:** <report-type>

[Report content...]
```

## Report Types by Agent

### ua-tester → `UAT/`

**What:** User accessibility and UI behavior test reports

**Files:**
- `UAT/UAT-YYYY-MM-DD-vNN.md`

**Naming Convention:**
- Prefix: `UAT-`
- Date: `YYYY-MM-DD`
- Version: `vNN` (resets daily)

**Metadata Required:**
- Report ID
- Date
- Time (with timezone)
- Agent name (ua-tester)
- Type (User Accessibility Test)

**When Created:** After manual UI testing or accessibility review

**Example:**
```
UAT/UAT-2025-11-28-v01.md
UAT/UAT-2025-11-28-v02.md
```

---

### test-runner → `testing/`

**What:** Test execution summaries, coverage metrics, test health trends

**Files:**
- `testing/test-runs.md` - Running summary of all test executions
- Detailed logs stored in `tests/logs/tests-YYYY-mm-dd.json` (outside this directory)

**Naming Convention:**
- Summary file: `test-runs.md` (single file, continuously updated)
- Individual reports (if needed): `testing-YYYY-MM-DD-vNN.md`
- Log files: `tests/logs/tests-YYYY-mm-dd.json` (daily logs)

**Metadata Required:**
- Timestamp of test run
- Command executed
- Pass rate and duration
- Link to detailed logs

**When Created:** After every test run via `pnpm test`, `pnpm test:unit`, etc.

## General Rules

1. **Flat structure**: No date subdirectories - all files in agent directory
2. **Daily versioning**: Version numbers reset each day (v01, v02, v03)
3. **Never overwrite**: Always increment version, never replace existing files
4. **Consistent naming**: Always use `<AGENT>-YYYY-MM-DD-vNN.md` format
5. **Agent ownership**: Only the designated agent creates/updates files in its area
6. **Metadata required**: Every report must include Report ID, Date, Time, Agent, Type
7. **Two-digit version**: Use v01, v02 not v1, v2
